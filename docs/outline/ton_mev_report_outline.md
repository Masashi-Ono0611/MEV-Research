# TON / TON<>EVM MEV 実情レポート＋解決策（アウトライン・ドラフト）

## 0. 目的と読者像
- 目的: TONおよびTON<>EVMクロスチェーンでのMEV実態を整理し、課題と対策を初学者にもわかりやすく提示する。
- 読者: MEV研究入門者、TON/Telegramエコシステム開発者、クロスチェーン運用者。

## 1. 背景と問題設定
- MEVの定義と代表パターン（アービトラージ、サンドイッチ、遅延/再org攻撃）。
- TONのアーキテクチャ概要（高スループット、手数料設計、ビルダー分離の有無）。
- TON<>EVMブリッジの特徴とリスク（遅延攻撃、価格乖離、再org耐性）。
- 参考: 他L1の関連研究（Ethereum PBS/MEV-Boost、SUAVE、Flashbots論文、Solana/Seiの順序設計、Aptos/Suiの並列実行とMEV議論）。

## 2. 実態調査の設計（データと指標）
- データ収集: トランザクション/ブロック/ブリッジイベント、価格フィード、DEXスワップログ。
- 指標: 検閲耐性（到達遅延/未収録率）、サンドイッチ被害額、スリッページ分布、レイテンシ格差、ビルダー集中度。
- 測定手法: サンドイッチ検知アルゴリズム、ブリッジ遅延計測、オークション支払ログの解析。

## 3. ケーススタディ（TON内）
- DEX/支払いユースケースでのMEV機会の分布。
- 手数料の低さがフロントラン/スパムに与える影響。
- ビルダー/バリデータ集中度と順序付けの実態。

## 4. ケーススタディ（TON<>EVMブリッジ）
- 典型的なクロスチェーンMEVパターン（遅延・再orgを狙った攻撃、価格乖離型アービトラージ）。
- 経路別リスク比較（公式ブリッジ vs サードパーティ）。
- 価格影響とユーザ被害額の簡易推定。

## 5. 解決策のカタログと評価軸
- オークション/PBS/ビルダー分離：透明化と配分調整。
- 有害抽出抑制：サンドイッチ拒否フラグ、暗号注文/バッチ、UX警告。
- ブリッジ防御：遅延検知、ファイナリティ強化、リスクプレミアム反映。
- 評価軸: ユーザ便益、実装コスト、レイテンシ影響、中央化リスク、プライバシー影響。

## 6. 推奨パス（TON/クロスチェーン向け初期提案）
- 短期: サンドイッチ警告＋フェア注文ルート提供、遅延監視と健全度ダッシュボード。
- 中期: オークション/PBS型の透明化とリベート設計の試験導入。
- 長期: ビルダー分離/PBS本格導入、ブリッジ安全性強化（ライトクライアント/再org耐性）と経路多様化。

## 7. オープン課題と今後の研究
- 有害判定の偽陽性/偽陰性トレードオフ。
- プライバシーと透明性のバランス。
- 分散度と性能（レイテンシ/スループット）の最適化。
